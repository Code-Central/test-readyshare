With our GitHub account created, we want to make a new repository to host our website!

We are going to be using GitHub's "GitHub Pages". There are some limitations to this, but for this course, it will work just fine!

First press the "+" button in the top right corner and select "New repository":

Give the new repository a name:

Initialize the README (this is to allow us to use GitHub Pages and host our website):

Create a repository:

Our new repository!

We want to upload the website files we created earlier into this new repository.

Click on "Upload Files":

We can drag our files directly into this section:

Open up your project folder in "File Explorer" (if you can't find it, ask an instructor for help), and drag everything into this section:

A bunch of files will have been uploaded:

Scroll to the bottom and add some comments on the "Commit Changes", and then press the "Commit changes" button when finished:

Wait until all the files are processed into the repository. With the files uploaded and processed, we need to activate "Pages" so that we can access it.

Press on the "Settings" tab:

Scroll down until you see the "GitHub Pages" section:

Set the "Source" to the "Master branch":

Your page will be reloaded, when you scroll back down you will see that your website is "Ready to be published":

After a few minutes, refresh the page and it should show a green message that says "Your site is published":

Click on the link and see your website!

That's your website! On GitHub!

All website projects you make will be required to be uploaded to GitHub before being marked completed.

Keeping an active GitHub account is vital in coding, this makes you the super cool kid on the block!

Congratulations!
